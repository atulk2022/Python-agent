## File Details & Descriptions

### Core Processing Engine

**src/fnol_processor.py** (450+ lines)
- **Purpose**: Main processing engine for FNOL documents
- **Key Classes**:
  - `FNOLProcessor`: Orchestrates extraction, validation, and routing
  - `ExtractedFields`: Data class for 18 insurance fields
  - `ClaimRoute`: Enum for 4 routing destinations

- **Key Methods**:
  - `process_document(file_path)`: Main entry point
  - `_extract_fields(content)`: Regex-based field extraction
  - `_identify_missing_fields(extracted)`: Validation
  - `_determine_route(extracted, missing, content)`: Routing logic
  - `_check_fraud_indicators(content)`: Fraud detection

- **Features**:
  - 18 field extraction patterns
  - 8 mandatory field validation
  - 5 fraud detection keywords
  - 4 routing rules with priority
  - JSON serialization support

### Execution Scripts

**test_runner.py** (150+ lines)
- **Purpose**: Batch process all FNOL documents
- **Functionality**:
  - Processes all FNOL_*.txt files
  - Generates individual JSON results
  - Creates summary report
  - Displays formatted console output
  - Counts routing distribution

- **Usage**: `python test_runner.py`
- **Output**: 6 JSON files + console report

**demo.py** (200+ lines)
- **Purpose**: Interactive demonstrations
- **Demonstrations**:
  - Single document processing with field extraction
  - Fraud detection capabilities
  - Routing comparison across all documents
  - JSON output format showcase
  - Missing field detection

- **Usage**: `python demo.py`
- **Output**: Console-based interactive tour
